# ts 缓存策略
## 原则
项目中可以提前缓存若干 ts 分片，供给播放器在播放时使用。

- 缓存存储在磁盘中
- 初始化缓存队列数据为空，等待播放器来取缓存时，再做缓存方面的添加或者修改的逻辑处理。
- 但是如果播放器在请求指定分片的缓存时，当前分片对应的缓存下载任务尚未下载完成，则放弃对于当前缓存的使用，播放器直接重新下载当前分片，指定分片的缓存任务将会被取消。
- 使用改写后的 EXT-X-MEDIA-SEQUENCE 来标识每个 ts 的序列号。
- 播放器触发读取序列号为 `X` 位置的缓存时，程序中应同时将 `X+1` `X+2` `X+3` 三个分片加入到缓存任务中，任务并行执行；同时清除 `< X` 的分片缓存的任务和磁盘数据。
- 更新 m3u8 列表时，如果已经知道最近一次播放的位置 `X`，则重做上一条逻辑。
- 切换源时，清空缓存队列和缓存磁盘数据。
